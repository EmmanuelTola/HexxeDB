XQL (Hexxe Query Language) Documentation

Overview

Welcome to the official documentation of XQL, a dynamic JavaScript-based query language designed for HexxeDB, a robust frontend database manager. This comprehensive guide will explore XQL from its basic syntax to advanced features, providing developers with powerful tools for efficient frontend data management.

Getting Started

Setup

Setting up XQL is straightforward. Install it on your localhost or file manager, create an account, and configure your tables and databases.

Initiate

let $con = "yourhexxedbpasskey"; $START('XQL:'+$con);

Syntax and CRUD Operations

Let's explore the fundamental syntax of XQL and delve into CRUD operations, laying the foundation for effective interaction with XQL.

Basic Query Operation

const $con = "yourhexxedbkey"; const $db = "your database"; $QUERY($con, $db, `SELECT users`).pull(capSule);

  • $con: Represents your HexxeDB key, providing access to your specific database.

  • $db: Specifies the database you want to interact with.

  • $QUERY: The fundamental function for querying data in XQL.

  • .pull(capSule): Initiates the handling of the data result. Customize the function (here) to process and utilize the retrieved data.

    • CRUD Operations

    Create (Insert) Operation

    $INSERT($db, 'INSERT users FIELDS(name, id) VALUES(name, id)', actionOnComplete*, actionOnError* ); // INSERT - Initiates the insertion of data into the specified table (users in this case) // .FIELDS(name, id) VALUES(name, id) - Defines the fields and corresponding values to be inserted.

    // form element names is name of field to insert

    Read (Query) Operation

    Basic Read

    QUERY($con, $db, `SELECT users`).pull(capSule);

    Filtering Query

    $QUERY($con, $db, `SELECT users WHERE name = 'Eric'`).pull(capSule); $QUERY($con, $db, `SELECT users REGEX name = 'r'`).pull(capSule); $QUERY($con, $db, `SELECT users COUNT name = 'Eric'`).pull(capSule); // returns number of results $QUERY($con, $db, `SELECT users INDEX name = 'Eric'`).pull(capSule); // returns index of result in entire table

    Update Operation

    $UPDATE($db, `UPDATE users SET name = 'newName' WHERE id = '1'`, actionOnComplete*, actionOnError*); // UPDATE - Initiates the update of data in the specified table (users in this case) // SET name = 'newName' WHERE id = '1' - Specifies the fields and conditions for the update.

    // form element names is name of field to update

    Delete Operation

    $DELETE($db, `DELETE users WHERE id = '1'`, actionOnComplete*, actionOnError*); // DELETE - Initiates the deletion of data from the specified table (users in this case) // .WHERE id = '1' - Specifies the conditions for deletion.

    Tools & Operation

    loop() - Iterating through HTML

    QUERY($con, $db, `SELECT users`).pull(useData); function useData(data) { let div = getHTML(target); for(const row of data) { row['color'] = randomColor(); row['bio'] = $nl2br(row['bio']); } let result = loop(data, div); // Iterate through data target.innerHTML = result; } // loop() - Iterates through HTML content with all available results. // In the example, it processes each row of data, adding a random color and formatting the 'bio' field with line breaks.

    $return() - Iterating Through HTML Content for Specific Result

    function useData(data) { let div = getHTML(target); let row = data[0]; row['color'] = randomColor(); row['bio'] = $nl2br(row['bio']); let result = $return(row, div); target.innerHTML = result; } // return() - Iterates through HTML content for only the specific result. Uses only first row of data

    $nl2br / nl2br

    function useData(data) { for(const row of data) { row['bio'] = $nl2br(row['bio']); } } // XQL escapes line breaks. This function is used to restore those line breaks. // $nl2br is innerHTML compatible while nl2br is innerText compatible

    $Condition();

    !!if[[name == Eric]][[ show this content]] else [[ show this content ]]!!

    function useData(data) { let result = loop(data, getHTML('#target')); result = $Condition(result); document.getElementById('target') .innerHTML = result; } // display content based on field conditions

    Complex Operations using $WHERE() - Speed Optimization

    $QUERY($con, $db, `SELECT users`).pull(useData); function useData(data) { $WHERE(data, `REGEX name = 'Eric'`).pull(usageOne); $WHERE(data, `COUNT name = 'Eric'`).pull(usageTwo); let currId = localStorage.getItem("uid"); $WHERE(data, `WHERE userid = '${currId}'`).pull(usageThree); } // During instances where same data is needed severally, instead of calling multiple queries, optimize performance by calling it once and using the data on other occasions using $WHERE

    $encode()

    function useData(data) { for(const row of data) { row['bio'] = $encode(row['bio']); } } // XQL escapes double and single quotes ("" & ''). This function is used to restore those them.

    Error Handling

    Traditional Method

    QUERY($con, $db, `SELECT users`).pull(useData, handleError); function handleError(error) { console.error("An error occurred: ", error); }
